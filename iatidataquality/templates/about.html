About

Publish What You Fund has developed this Data Quality tool to assess the quality of aid information published in IATI XML format. It is designed to run a series of automated checks and tests on data published in IATI XML format and present findings in a way that provides both a comparative view across all organisations and granular level details on each organisation’s data and any gaps.

Purpose

Several organisations have started publishing information on their aid spending to the International Aid Transparency Initiative (IATI) Registry. While a number of tools have been developed to explore, access and visualise data published to the Registry, at present, it is difficult to obtain an overall picture of the quality of the data being published to the Registry.

It is generally recognised by IATI publishers that there is room for improvement in the type and coverage of information being published to the Registry. In some cases, this is because of relatively minor problems inherent in the process of publishing information on large volumes of data, while in others, it relates to bigger problems of non-delivery on commitments and implementation schedules.

Given that the number of publishers is likely to rise significantly as organisations begin to publish in line with their commitments under the common standard, there is a clear need for a mechanism to provide feedback on the quality of the data to ensure that the ultimate purpose behind publishing to IATI is met – making timely, comprehensive, comparable and forward looking information on development cooperation available to all stakeholders. The Data Quality too has been developed in response to this need.

Methodology

The quality of data published in IATI XML is assessed by running a series of tests on all activity and organisation data packages currently being published to the Registry. These tests have been designed to assess the availability, comprehensiveness and comparability of the information and to determine whether an organisation’s publication conforms to the IATI standard appropriately. Most of the tests have been derived directly from IATI schemas provide formats for reporting data on various fields to the IATI Registry. Some additional tests have been designed to ensure that data published in IATI XML is done in a manner which allows for comparison across donors.

For the purposes of the 2013 Index, tests are run against only those activities with planned or actual end dates within the previous 13 months. This is to ensure that the information gathered relates to the period being assessed in the Index, i.e. information published on or after April 1, 2012 or that relates to that period. These same tests can also be run against all IATI data published by an organisation. The Data Quality tool adopts an expansive approach to testing, whereby the tests run on the data are flexible and can be expanded in the future to potentially find out a large amount of comparative information about IATI data.

For a complete list of the tests being run for determining indicator scores for the 2013 Index, visit the tests section of the Data Quality tool.

Results and aggregation

The tool returns the following results for each test run on a given data package:

  1. A pass or fail result for each file within a data package;
  2. Percentage of activity or organisation files within a data package that have passed the test.

The results also contain information on:

  1. The test ID;
  2. The data package ID;
  3. The runtime ID;
  4. The IATI activity identifier (where applicable)

Test results are currently aggregated – based on the 2013 Aid Transparency Index indicators they pertain to – in order to produce scores for these indicators. The tests underlying the measurement of each indicator can be viewed by clicking on the indicator names in the publication page for the relevant organisation. Survey methodology

Surveys for all organisations included in the 2013 Aid Transparency Index are completed using information relating to the country that received the largest amount of aid by value from that donor organisation. The OECD DAC's 2011 CRS data is used for determining the largest recipient. If this information is not available in the CRS, then the largest recipient is determined using the latest annual report for the donor organisation or related ministry. To establish that the information is consistently published at the activity level, a minimum of five activities are selected at random in that country. While checking and verifying data, organisations are asked to confirm if the responses are representative as a whole.

The survey tool is designed to capture the format (machine readable XML or CSV, webpages or PDF) in which the information is provided and whether it is 'always' published, 'sometimes' published or 'not published'.
